# COCoLa サイト GitHub Pages 移行設計

## 目的

現行の Google Sites ベースの COCoLa サイトを、デザイン自由度と保守性の高い GitHub Pages ベースの静的サイトへ再構成する。

この設計では、以下を両立する。

- 現行 Google Sites のページ構成を大きく崩さない
- 既存の自動化済み Web アプリ群を壊さず接続する
- LP と各詳細ページの役割を分けて、初見の人にも伝わる導線にする

## 前提

- 現行トップ: `https://sites.google.com/view/cocola-project/home`
- 既存の外部機能:
  - クリエイターズマップ
  - 文化財マップ
  - イベント一覧
  - おでかけMAP
- これらは今後も独立した公開URLを持つ前提で扱う
- `cocola_lp/` は新トップページ試作の作業ディレクトリ

## 基本方針

### 1. サイト全体を静的サイトとして再編成する

Google Sites の「ページ管理」を、GitHub Pages 用の HTML ファイル群に置き換える。

### 2. トップページは LP として作る

トップページは一覧的な情報集約ではなく、

- COCoLaとは
- 目的と事業内容
- 活動の雰囲気
- 最新トピックス
- 参加導線

を伝える入口ページとして設計する。

### 3. 6つの「つくる」は個別ページ化する

Google Sites のサイドバー構成に相当する形で、6分類ごとのページを個別に持つ。

### 4. 自動化済み機能は独立運用のまま接続する

既存の Web アプリは GitHub Pages に移植せず、まずは以下のいずれかで接続する。

- 通常リンク
- `iframe` 埋め込み

初期段階では、影響を最小化するため通常リンク優先とする。

## ページ構成案

### 1. トップページ

役割:

- 初見ユーザーに COCoLa の魅力を伝える
- 参加・問い合わせへつなげる
- 主要ページと主要アプリの入口になる

主なセクション:

- ヒーロー
- COCoLaとは
- 目的と事業内容
- 6つの「つくる」概要
- 活動の雰囲気
- 動画
- 最新トピックス
- 参加 CTA

### 2. 6つの「つくる」ページ

各ページで持つ内容:

- カテゴリの説明
- 関連する活動例
- 写真
- 関連トピックス
- 関連する外部機能や募集導線

対象:

- ものをつくる
- まちをつくる
- つながりをつくる
- 食をつくる
- 自分をつくる
- ともにつくる

### 3. トピックスページ

役割:

- Google Sites に散在していた更新情報や活動記録の一覧化
- LP では拾いきれない詳細情報の受け皿

### 4. アプリ入口ページ

役割:

- クリエイターズマップ
- 文化財マップ
- イベント一覧
- おでかけMAP

を整理して見せるハブページ

## 既存自動化アプリの扱い

### 方針

初期移行では、既存アプリのコードや公開方法は変更しない。

### 接続方法

- トップまたはアプリ入口ページからリンク
- 必要に応じて個別ページ内に埋め込み

### 注意点

- Google Sites 特有の埋め込み設定に依存していないか確認する
- `?embed=1` などの表示切替パラメータがあるものは維持する
- GitHub Pages 側では高さ・幅・レスポンシブ調整のみ行う

## デザイン方針

- COCoLa ロゴの色を基調としたカラフルで親しみやすいUI
- 実写写真を活かして「人が見える」サイトにする
- トップは LP、詳細は読み物ページとして役割分担する
- フォントはメイリオ基準で可読性重視

## 今後やるべき事項

### 優先度: 高

1. 現行 Google Sites の全ページ構成を棚卸しする
2. 6つの「つくる」各ページに載せる正式文言と素材を集める
3. 自動化済み 4アプリの正式公開URL一覧を確定する
4. GitHub Pages 用のリポジトリ名と公開URL方針を決める
5. 現行 LP 試作をトップページとして正式化するか判断する

### 優先度: 中

1. 共通ヘッダー・フッター・サイドバーを部品化する
2. 6つの「つくる」ページのテンプレートを作る
3. トピックス一覧の更新方法を決める
4. メンバー紹介ページとの導線を整理する
5. 動画・写真・ロゴの利用素材を `assets/` に整理する

### 優先度: 低

1. Apps 入口ページをつくる
2. 埋め込み版とリンク版の使い分けを最適化する
3. OGP画像、favicon、SEOメタを整備する

## リスクと注意点

- Google Sites 上の既存リンク資産が切れる可能性がある
- 個別ページを先に作らずにトップだけ先行公開すると、導線の空リンクが発生しやすい
- 外部アプリのURLや埋め込み条件が未整理だと移行後に表示差分が出る

## 推奨移行ステップ

1. まず GitHub Pages 上で新トップを公開する
2. 現行 Google Sites から新トップへ案内を出す
3. 6つの「つくる」ページを順次追加する
4. トピックスとアプリ導線を整理する
5. 最後に Google Sites 依存を外す
