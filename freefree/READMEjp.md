# freefree マイページの「メール切替できない問題」について

## ❓ よくある誤解：「ブラウザのキャッシュが原因？」

**いいえ、違います。**

「キャッシュをクリアすれば直る」と思っている人が多いのですが、本当の原因は別のところにあります。

---

## 🔍 本当の原因（中学生でもわかる説明）

### ブラウザは「メモ帳」を持っている

ブラウザ（Chrome や Safari）は、サイトごとに**小さなメモ帳**を持っています。
このメモ帳のことを `sessionStorage` といいます。

### freefreeマイページの仕組み

1. 初めてマイページを開くと → ログイン画面が表示される
2. メールアドレスを入力してログイン → ブラウザのメモ帳に **「myEmail = 〇〇@example.com」** と書き込まれる
3. 次にマイページを開くと → メモ帳を見て「あ、〇〇さんだ！」と判断 → **自動でログイン状態になる**

### 問題はここから

別の人（または同じ人の別アカウント）でログインしたいとき:

- メモ帳に前のメールアドレスが残っている
- ページを開くたびにメモ帳が読まれる
- **どうやっても前のアカウントで開いてしまう**

これが「キャッシュをクリアしないと別のメールで入れない」と言われる現象の正体です。

---

## ✅ 対策

### メモ帳を消すボタンを設置する

**メモ帳（`sessionStorage` の `myEmail`）を消すボタン**を画面に追加するだけで解決します。

ボタンを押すと:

1. メモ帳の `myEmail` を消す
2. ページをリロードする
3. メモ帳が空になったので、**ログイン画面が再び表示される**
4. 別のメールアドレスでログインできる

---

## 💡 ポイント

| よくある誤解 | 正しい理解 |
|---|---|
| ブラウザのキャッシュが原因 | `sessionStorage` という小さなメモ帳が原因 |
| ブラウザの設定からキャッシュを全部消す必要がある | ボタン1つで `sessionStorage` の1項目を消すだけ |
| ブラウザを変える/シークレットモードで開く必要がある | 同じブラウザのまま、ボタンを押すだけ |

---

## 🔧 実装したもの

`mypage.html` に以下を追加しました:

### 画面上のボタン

「← 掲示板へ戻る」の右に、**🔄 ログアウト（別アカウントへ）** ボタンを配置。

### ボタンの動作（JavaScript）

```js
function logoutMypage() {
  if (!confirm('ログアウトして別のメールアドレスでログインし直しますか？')) return;
  sessionStorage.removeItem('myEmail');  // ← メモ帳の myEmail を消す
  location.reload();                      // ← ページを読み直す
}
```

たったこれだけで解決します。

## 📚 補足：用語の整理

- **キャッシュ（cache）**: 一度読み込んだ画像やファイルを保存しておいて、次回速く表示するための仕組み。ログイン情報とは別物。
- **sessionStorage**: ブラウザのタブを開いている間だけ覚えておくメモ帳。タブを閉じると消える。
- **localStorage**: 似たような仕組みだけど、タブを閉じても残るメモ帳。

freefreeマイページが使っているのは `sessionStorage` のほう。
だから本来は**「タブを閉じれば消える」**はずなのですが、タブを開きっぱなしで使うことが多いので、結果的に「ずっと残っている」と感じてしまうのです。
